Lesson 0.2 - Introduction to Programming

By Daniel Luna on August 8, 2018 | Last Modified by Daniel Luna on August 9, 2018


History

Computers are always improving by becoming faster and more powerful. If you would have told the enginners the built the Machine that took man to the moon that eventually all of that could be condensed into a watch (Smart Watch) they would have called you crazy. But, programming languages make a layer that allows all supported computer parts work under one terminology. Some examples include Java, C++, Swift, JS, etc. So knowing at least one is very powerful as some languages let you do so much with the computer. Furthermore, computers are made for numbers so you should use them as so. You can program Math applications which do things that takes humans hours and let the computer do it in a couple of minutes. Besides all that, lets get started on the history of Programming Languages.


Machine Language

A CPU can't read english or C++ or anything as a matter of fact. Machine Code is just '0s' and '1s' which is binary. Each instruction is a number of binary digits. Some architectures like x86 (32-Bit) or x86-64 (64-Bit) allow for variable length instructions. On the other hand, MIPS architecture had 32-Bit long instructions no matter what. Eventually, something would have to be done in order to make coding accessable to more people. Especially, because in the 50s and 60s one machines code might not work in another so they had to build uniform architectures and a layer on top. This would eventually become the first programming language.


Assembly Language

As, Machine Language was hard with, Assembly was created to make it easier to program. Every command had a shorten name instead of a number representaion. This one change made it easier to read and write. But, this made another problem in which computers don't know how to read Assembly. So a program was made that translated the programmers assembly into machine code. But, it still only worked on one architecture so we had to go even higher up.
